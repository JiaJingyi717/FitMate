## 遇到的问题和解决

1. **问题：**浏览器控制台无法直接调用 API 函数

​	**解决：** 最开始我尝试在控制台直接调用封装好的 API 函数，但发现函数未定义。后来意识到 Vue 组件中的函数默认是局部作用域，控制台无法访问。通过将 API 函数挂载到 `window` 对象上（如 `window.loginApi = login`），成功实现了在控制台中直接调用接口并查看返回数据。

2. **问题：**部分接口返回 401（未登录）

​	**解决：**在测试需要登录权限的接口时，经常出现 401 错误。经过分析发现是请求中没有携带 token。于是我在 axios 请求拦截器中统一添加 Authorization 请求头，并在登录成功后将 token 存入 localStorage，从而保证后续接口调用可以正常通过认证。


## 心得体会

​	通过本周的前端开发实践，我对前后端分离开发流程有了更加清晰的认识。首先，在没有后端的情况下，可以通过 Mock 技术模拟接口数据，使前端开发不受限制，提高开发效率。其次，通过封装 API 调用函数，可以使代码结构更加清晰、易于维护。

​	在调试过程中，我学会了利用浏览器控制台直接调用接口，这种方式相比编写页面测试更加高效，也更直观地反映接口是否正常工作。同时，通过处理 token 认证问题，我对接口鉴权机制也有了初步理解。
