## 遇到的问题和解决

1. **问题：** OpenAPI Schema 与前端设计稿中的字段不一致（注册多凭证、教练性格等）。  
   **解决：** 在 `docs/api.yaml` 中扩展注册/登录相关 Schema，将教练配置拆分为 `gender` + `personality` 等，并与 `docs/api.md` 中的路径说明保持一致，减少联调歧义。

2. **问题：** `training_record` 中 `plan_id` 若必填，用户临时自由训练无法落库。  
   **解决：** 将 `plan_id` 设为可空，以 `user_id` 为必填归属；在 `backend/services/plan_service.py` 与文档中说明「无计划」打卡语义。

3. **问题：** JWT 鉴权若写在各路由中易重复、难维护。  
   **解决：** 在 `backend/utils/extensions.py` 统一 `JWTManager`，路由使用 `@jwt_required()`，`get_jwt_identity()` 取用户 ID 后交给 service 处理业务。

## 心得体会

在 API 设计与实现过程中，深感「先立契约再写代码」的价值：先在 `docs/api.yaml` 与 `docs/api.md` 中固定路径与响应结构，再实现 `routes` / `services`，返工更少。后端按蓝图拆分路由、业务下沉到 service、统一 `ok` / `fail` 响应，既便于 pytest 覆盖（当前仓库共 45 个用例），也方便后续扩展真实 AI 接口而不打乱整体结构。
