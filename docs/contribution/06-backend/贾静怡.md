## 遇到的问题和解决

1. **问题：** 用户注册需要支持多种凭证（邮箱/手机/用户名），但需确保唯一性。  
   **解决：** 在 `auth_routes.py` 中先检查 email/phone 是否已存在，然后根据提供凭证自动生成唯一 username（邮箱取 `@` 前缀 + hash 随机后缀），避免冲突。

2. **问题：** 忘记密码 / 重置密码涉及验证码，而当前无真实短信/邮件服务。  
   **解决：** 实现为模拟接口（验证码硬编码为 `123456`），返回 `ok` 响应便于前端流程调试，待接入真实服务时只需修改 `forgot_password` / `reset_password` 逻辑。

3. **问题：** Docker 部署时 MySQL 服务启动慢，导致 Flask 容器连接失败。  
   **解决：** 在 `docker-compose.yml` 中添加 `healthcheck` + `depends_on: condition: service_healthy`，确保 MySQL 完全就绪后再启动应用容器。

## 心得体会

在后端开发过程中，深刻体会到「服务分层」的重要性：routes 只做参数解析与鉴权，数据库操作下沉到 services，工具函数抽离到 utils，使得代码结构清晰、易于测试。JWT 鉴权统一使用 `@jwt_required()` + `get_jwt_identity()`，避免了在各路由中重复编写认证逻辑。Docker 容器化部署让我学会了服务编排与健康检查机制，虽然是模拟的验证码功能，也为后续接入真实第三方服务预留了扩展接口。整个后端体系（认证、用户、成就、部署配置）已经可以作为可复用的 Flask 模板。
