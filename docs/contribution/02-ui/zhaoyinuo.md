## 遇到的问题和解决

**1. 问题：页面功能较多，信息层级容易混乱**
 解决：通过信息架构设计，将系统主要功能划分为 AI 教练、训练计划、知识库、数据分析和个人中心五个模块，使整体结构更加清晰。

**2. 问题：UI设计最初无从下手**
 解决：借助Figmamake辅助生成的初版架构，再此基础上进行思路完善，并手动更改完善设计，同时在b站上进行了学习如弹窗、组件、悬停等教程。

​	在本次 UI 设计过程中，我学习并实践了使用 Figma 进行界面设计的方法，包括页面布局设计、组件管理以及设计规范制定等。同时，通过对用户需求和系统功能的分析，使界面结构更加清晰、逻辑更加合理。同时页面设计的具体性也让我反思功能设计的可行性，从而完善功能上的逻辑性。通过本次设计，我进一步理解了 UI 设计在软件开发过程中的重要作用，也提高了自己在界面设计和产品思考方面的能力。
