## 遇到问题与解决

1. 问题：页面组件在 `onMounted` 中请求数据，测试中 `mount` 后直接断言发现数据尚未加载完成，导致测试失败。

   解决：使用 `flushPromises()` 等待所有异步操作完成后再进行断言，确保组件真正渲染完成。

2. 问题：组件中调用 `window.confirm()` 用于删除确认，测试环境报 `window.confirm is not a function`。

   解决：在 `beforeEach` 中使用 `vi.fn(() => true)` 手动注入 Mock 实现。

3. 问题:多个 `it` 测试用例复用同一个 Mock 函数时，前一个用例的调用记录会影响后续用例的断言结果。

   解决：在 `beforeEach` 中对所有 Mock 函数调用 `mockReset()` 重置状态，并在测试前清理 `localStorage` / `sessionStorage`。
